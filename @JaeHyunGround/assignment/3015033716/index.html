기본과제

가상돔을 기반으로 렌더링하기

  • createVNode 함수를 이용하여 vNode를 만든다.
  • normalizeVNode 함수를 이용하여 vNode를 정규화한다.
  • createElement 함수를 이용하여 vNode를 실제 DOM으로 만든다.
  • 결과적으로, JSX를 실제 DOM으로 변환할 수 있도록 만들었다.

이벤트 위임

  • 노드를 생성할 때 이벤트를 직접 등록하는게 아니라 이벤트 위임 방식으로 등록해야 한다
  • 동적으로 추가된 요소에도 이벤트가 정상적으로 작동해야 한다
  • 이벤트 핸들러가 제거되면 더 이상 호출되지 않아야 한다

심화 과제

1) Diff 알고리즘 구현

  • 초기 렌더링이 올바르게 수행되어야 한다
  • diff 알고리즘을 통해 변경된 부분만 업데이트해야 한다
  • 새로운 요소를 추가하고 불필요한 요소를 제거해야 한다
  • 요소의 속성만 변경되었을 때 요소를 재사용해야 한다
  • 요소의 타입이 변경되었을 때 새로운 요소를 생성해야 한다

과제 셀프회고

아하! 모먼트 (A-ha! Moment)

과제를 하면서 초반에는 createVNode와 normalizeVNode 함수의 차이가 무엇인지 이해가 되지 않았습니다. 두 함수 모두 가상 DOM JS 객체를 return하는 함수였기에 동일한 역할을 한다고 생각했습니다. 하지만 내부 로직을 살펴보면서 두 함수는 비슷하면서도 명확하게 역할이 구분되어 있다는 것을 이해할 수 있었습니다.

createVNode 함수는 JSX를 가상 DOM JS 객체로 변환하는 가상 노드 생성 역할을 하고, normalizeVNode 함수는 가상 노드를 정규화하는 역할을 합니다. 가령 객체의 children으로 함수형 컴포넌트가 포함되어 있을 때 이를 적합한 JS 객체로 변환해야 할 때 normalizeVNode 함수가 진가를 발휘하는 것 같았습니다. (createVNode => React.createElement() 와 동일한 역할, normalizeVNode => React 재조정(Reconciliation) 과정 중 일부에 해당)

이외에 createElement 함수로 JS 객체 => 실제 DOM 요소 생성, 이벤트 위임을 통한 성능 향상, diff 알고리즘을 활용하여 변경되는 부분만 교체해줌으로써 렌더링 성능을 향상 시킨다는 개념은 이해했으나 함수의 내부 로직은 정확하게 이해하지 못한 것 같습니다. 해당 주차가 끝나더라도 easy 내용 뿐만 아니라 hard 과제도 도전해보며 이전까진 해보지 못했던 내가 사용하는 라이브러리 뜯어보기 경험을 해보고 싶습니다.

학습 효과 분석

추상적으로만 알고 있던 가상 DOM을 활용한 렌더링 정책에 대해 기술적인 개념을 학습할 수 있었습니다.

가상 DOM의 개념에 대해 DOM의 복사본 정도로 알고 있었는데, 가상 DOM은 DOM의 형태를 본따 만든 JS 객체라는 것을 알 수 있었고, diff 알고리즘의 개념과 이벤트 위임 등 React가 렌더링 최적화를 위해 어떤 기법을 사용하였는지 파악할 수 있었습니다.
