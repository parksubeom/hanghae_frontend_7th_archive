과제 셀프회고

늘 그렇듯 쉽다고~ 쉽다고~ 하시지만 까보면 어려움 천지라 "이번에도 밤새겠네~" 했는데, 이게 웬걸? 수요일에 과제를 끝내버렸다... 과제를 하면서도 "이거 이렇게 하는게 맞나..?"라는 생각이 역대급으로 많이 들었다. 진짜 "다음에 뭘 해야 해결되지?"의 의미도 있지만, "이거 이렇게 간단하게 끝내는게 맞나..?"가 압도적으로 많이 느껴졌다. 그래서 정말정말 쉬웠지만, 문제는 뒷걸음질치다 쥐잡는 꼴처럼 "내가 진짜 의도해서 해결한건가?"와 "이 의도가 맞는 걸까?"가 많이 의심됐다. 일단은 해결은 했지만 PR을 적으면서 내가 한 방식이 맞는 지에 대해 회고해보고자 한다.

기술적 성장

이제는 친해졌다 React..!

사실 10주동안 내내 리액트를 써왔지만, 나의 주 무기는 Vue.js이다 보니, 사용에 있어 어색한 부분이 많다고 느꼈다. 특히 메모이제이션 파트는 Vue에선 생소한 영역이라, 개념적으로는 이해했지만 "언제 써야하지?", "여기에 적용시키는게 맞나?"에 대한 확신이 잘 안들었다. 마침 이번 과제에서 렌더링 최적화를 하다보면 자연스럽게 메모이제이션을 적용시켜야하는데, 그 과정에서 "어느 범위까지 적용시켜야 하는지", "어떨때 해야하는지"에 대해 확실히 감을 잡게 된 것 같다. 물론 무작정 컴포넌트 쪼개고 memo로 감싸고 props로 내리는 이벤트핸들러에 useCallback 적용시키고 등등.. 뚜렷하게 분석된 결과물은 아니긴 하지만..!이제서라도 그 감을 확실히 느끼게 된게 가장 큰 수확인 것 같다.

학습 효과 분석

이전에는 메모이제이션 훅들을 '감'에 의존해 사용했다면, 이번 과제를 통해 React DevTools 프로파일러로 렌더링 병목을 직접 확인하고, 각 훅이 왜 필요한지, 언제 사용해야 하는지를 이해하게 된 것 같다. 그리고, Q&A에서 상태와 액션을 분리하는 Context 패턴이 왜 필요한지, 그리고 이것이 어떻게 불필요한 리렌더링을 막아주는지 원리를 이해한 것이 가장 큰 학습 효과라 생각한다.

과제 피드백

React DevTools 프로파일러를 통해 리팩토링 전후의 렌더링 차이를 직접 확인하면서 과제를 한게 기억에 남는다. 최적화의 효과를 즉각적으로 피드백 할 수 있어서 수월하게 과제를 진행할 수 있었다. 또한 컴포넌트 분리, Context 최적화 등 이번 과제에서 다룬 내용들은 실제 현업에서 자주 마주치는 성능 문제를 해결하는 데 직접적으로 도움이 될 실용적인 기술들이라 유익했다고 느껴진다.
