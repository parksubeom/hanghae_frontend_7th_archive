과제 체크포인트

필수 스펙

    1. 반복 유형 선택
    • 일정 생성 또는 수정 시 반복 유형을 선택할 수 있다.
    • 반복 유형은 다음과 같다: 매일, 매주, 매월, 매년
      • 31일에 매월을 선택한다면 → 매월 마지막이 아닌, 31일에만 생성하세요.
      • 윤년 29일에 매년을 선택한다면 → 29일에만 생성하세요!
    • 반복일정은 일정 겹침을 고려하지 않는다.
  1. 반복 일정 표시
    • 캘린더 뷰에서 반복 일정을 아이콘을 넣어 구분하여 표시한다.
  2. 반복 종료
    • 반복 종료 조건을 지정할 수 있다.
    • 옵션: 특정 날짜까지
      • 예제 특성상, 2025-12-31까지 최대 일자를 만들어 주세요.
  3. 반복 일정 수정
    1. ‘해당 일정만 수정하시겠어요?’ 라는 텍스트에서 ‘예’라고 누르는 경우 단일 수정
      • 반복일정을 수정하면 단일 일정으로 변경됩니다.
      • 반복일정 아이콘도 사라집니다.
    2. ‘해당 일정만 수정하시겠어요?’ 라는 텍스트에서 ‘아니오’라고 누르는 경우 전체 수정
      • 이 경우 반복 일정은 유지됩니다.
      • 반복일정 아이콘도 유지됩니다.
  4. 반복 일정 삭제
    1. ‘해당 일정만 삭제하시겠어요?’ 라는 텍스트에서 ‘예’라고 누르는 경우 단일 수정
      1. 해당 일정만 삭제합니다.
    2. ‘해당 일정만 삭제하시겠어요?’ 라는 텍스트에서 ‘아니오’라고 누르는 경우 전체 수정
      1. 반복 일정의 모든 일정을 삭제할 수 있다.

# AI와 테스트를 활용한 안정적인 기능 개발 리포트

## 사용하는 도구를 선택한 이유가 있을까요? 각 도구의 특징에 대해 조사해본적이 있나요?

## 테스트를 기반으로 하는 AI를 통한 기능 개발과 없을 때의 기능개발은 차이가 있었나요? 

## AI의 응답을 개선하기 위해 추가했던 여러 정보(context)는 무엇인가요? 

## 이 context를 잘 활용하게 하기 위해 했던 노력이 있나요?

## 생성된 여러 결과는 만족스러웠나요? AI의 응답을 어떤 기준을 갖고 '평가(evaluation)'했나요?

## AI에게 어떻게 질문하는것이 더 나은 결과를 얻을 수 있었나요? 시도했던 여러 경험을 알려주세요. 

## AI에게 지시하는 작업의 범위를 어떻게 잡았나요? 범위를 좁게, 넓게 해보고 결과를 적어주세요. 그리고 내가 생각하는 적절한 단위를 말해보세요.

## 동기들에게 공유하고 싶은 좋은 참고자료나 문구가 있었나요? 마음껏 자랑해주세요.

## AI가 잘하는 것과 못하는 것에 대해 고민한 적이 있나요? 내가 생각하는 지점에 대해 작성해주세요.

## 마지막으로 느낀점에 대해 적어주세요!

과제 초반

  • 주말시간을 활용해서 발제자료 + 참고자료 읽음 (읽긴 하는데 머리에 잘 안들어왔습니다,,, 영어는 너무 많다)
  • PRD는 뭐지? 명세서라는데, 이미 노션에 명세서를 주신게 아닌가? 잘 만든 PRD문서가 따로 있나? => 약간 바보같지만, 저는 '명세서'의 사전적 의미는 알아도 실제 개발을 할 때 명세서를 가지고 개발해본적이 없어서 정말 처음 agent 만드는 것부터 힘들었습니다ㅠ
  • 코파일럿 사용 (이유: vscode가 익숙하니까? 그리고 코파일럿이 저렴해서,, 10달러. 근데 gemini pro를 추가로 결제했어요.. 29달러 agent 만드려고)
  • 다들 BMAD-Method 형식을 따왔다길래, 저는 무작성 BMAD를 설치하고 적용해보려 했는데요, 잘 안되었습니다..!
    • 생각해보면 제가 원하는 출력 형식도 모른채, 좋다고 무작정 도입해서 그런것 같아요.
  • 초반 스타트를 무작정 과제 발제 내용과 프로젝트 구조 정보를 가지고, AI에게 던지면서 만들어줘! 라고 요구했습니다. Q&A를 듣고 아, Input -> Output부터 정하고 그 사이 과정을 LLM에게 맡겨야 하는구나 알았습니다. (꾸역꾸역 2개 agent 만들었는데, 다 갈아엎어야겠다..! 생각했습니다ㅠㅠ)

과제 중반

  1. 명세 / 기능을 구체적이게 작성하기 Agent
    • Q&A 듣고 다시 처음부터 에이전트 만들기 시작.
    • Input -> Output 형식부터 정함. Output 은 발제자료에 첨부하신 뱅크샐러드 테크노트 형식 참조함.
  2. 코파일럿에서 기본적으로 제공하는 gpt-5 mini 를 썼는데, 그냥 그런갑다했다. 무료버전분 Sonnet4.5를 좀 써봤는데 OMG 빠르기도 하고, 생성되는 결과 값이 확연히 달랐다. 바로 Sonnet4.5 API 결제했다. (최고)
    • Sonnet4.5 을 쓰니까 오히려 내가 입력한것 보다 훨씬 많은 결과물들을 팍팍 만들어줬다.
    • 내가 모호하게 작성하니까 오히려 얘가 못알아듣고, 더 많은 출력을 한 것 같다. 내가 원하는 출력방식을 명확히 지정하니, 4.0과 4.5의 출력값의 차이가 많이 줄어들었다.

과제 후반

  1. 테스트가 너무 오래걸린다ㅠㅠ 원래 이렇게 한번한번 돌리는데 오래걸리나요..??
  2. [문제발생!] GREEN 단계가 예상치 못한 복병이었다. (특히 통합테스트) 코파일럿 챗으로 티키타카 처리하려했더니 테스트에서 getByRole로 컴포넌트를 가지고 못하는 이슈가 생겼다. 떄문에 테스트 통과도 못하고 있다.. Sonnet4.5로 돌리는데, 코드작성 -> 실패 -> 원인분석 -> 코드작성 -> 실패 ... 무한루프같이 돈다.
    • 최대 반복횟수를 3번으로 정함. 타임아웃도 넣음.
    • 에러 출력된것을 보니까 테스트에서 MUI의 컴포넌트를 가지고 오지 못하는 문제가 발생했다 (ex. screen.getByRole('option', { name: '매일' })) 컴포넌트에서 Portal 을 써서 그렇다는데 이거를 어떻게 해결해야하지? 싶었다. (추후 원인을 파악해야할 것 같습니다) 다른 학메님도 저랑 같은 이슈가 발생해서 셀렉터를 써야하지 않을까요? 라고 팁을 주셨다.
  3. 명세를 잘 작성해도, RED Agent가 테스트 케이스 expect 값을 잘못 작성하는 경우가 있음을 발견했다. 테스트케이스를 내가 확인을 했어야 했는데 무조건 믿고 GREEN 테스트로 넘어가니까 잘 안된 것 같다. GREEN 단계에서 AI 가 꾸역꾸역 테스트케이스에 맞춰 구현하다보니 최대 반복 날짜 조건인 Max Date 2025-12-31 조건까지도 구현에서 빼버린 경우가 생겼다.

기술적 성장

  • [새로 배움]
    • AI Agent를 만들때도 함수처럼 생각해라! Input -> Output 을 먼저 생각하고, 그 사이 과정을 LLM이 채우도록 해라 (오프코치님 Q&A때 큰 깨달음을 얻었습니다)
    • BMAD-Method 라는것을 이번에 처음 알았습니다...!
    • 동일한 Agent 여도, LLM에 따라 결과값이 확연히 다름을 알았다. (ex. gpt5-mini VS Sonnet 4 VS Sonnet 4.5)
      • gpt5.1-mini -> Sonnet4.5 해봤더니 출력 결과도 매우 다르고 성능/속도 모두 다르다.
  • [지식의 심화]
    • 예시/출력 형식을 결정해서 넣으면, 내가 의도한 바대로 agent가 동작함.
    • 금요일 새벽, 젭에서 LLM별로 성격이나 출력 차이가 있다는 이야기를 들었다. 명세는 Gemini가 작성하게하고, 검증은 클로드가 하게 하는 등. 각 LLM 별 특징을 가지고 출력하는 방법도 있구나 싶음.
  • [에이전트 테스트] 같은 챗을 쓰면 캐시가 되는지, 이전 에이전트 md 내용이 나왔음. 코파일럿 계속 New Chat 만들어서 새로운 환경에서 독립적으로 에이전트 테스트를 했다.
