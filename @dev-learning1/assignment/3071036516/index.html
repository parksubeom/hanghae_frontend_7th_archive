과제 셀프회고

과제를 하면서 내가 알게된 점, 좋았던 점은 무엇인가요?

어떤 것을 상태관리 또는 props로 나누면 좋을 지에 대해 고민한 시간이었습니다. props drilling 이라는 용어를 처음 알게되었고, 기존에는 처음에 개발하는 것에 급급해 대부분을 props로 진행하다가 나중에 2단계 이상요소에서 필요하게 되면 context/zustand로 상태관리로 변경하여 사용했었습니다. 그런데 이번 과제에 대해 고민하면서 프로젝트 초반에 이러한 것에 대해 고민을 한 후 진행을 하면 더 효율적일 것 같다라는 생각을 했습니다.

이번 과제에서 내가 제일 신경 쓴 부분은 무엇인가요?

공통 컴포넌트와 페이지에서만 사용되는 컴포넌트 구분 예를 들어 상품을 나타내는 카드 컴포넌트를 만들어야 한다고 가정해보면... 카드 디자인, 즉 border, background color, font size/weight/color 등 일반적인 카드 디자인을 공통 컴포넌트로 만듬 ( 이때, 공통 컴포넌트에서 관리하는 컴포넌트들은 cva, cn 을 사용하여 커스텀할 수 있도록 코드 작성 해야함) 공통 컴포넌트의 카드 컴포넌트를 커스텀하여 페이지에 사용되어지는 상품 카드 컴포넌트를 생성 상품 카드 컴포넌트는 해당 페이지폴더의 components폴더에서 관리 이렇게 하면 카드 형식의 요소들은 모두 동일한 디자인을 갖게 되므로 도메인 화면에 통일성을 줄 수 있을거라 생각했음 (실제로 좋은 방식인지는 모르겠어요...)

src/app/
├── pages/           
│   └── shop/
│    |    └── components/  → shop 페이지에서 다루는 컴포넌트(공통 컴포넌트를 커스텀, 순수 작업한 컴포넌트)
│    |    └── page.tsx     
│   └── admin/    
│    |    └── components/  → admin 페이지에서 다루는 컴포넌트(공통 컴포넌트를 커스텀, 순수 작업한 컴포넌트)
│    |    └── page.tsx     
│
└── components/         → 공통 컴포넌트
│   └──  card.tsx        → 커스텀할 수 있는 일반적인 카드 컴포넌트(cva, cn 을 사용하여 커스텀 가능할 수 있게 작성)
└── page.tsx 
└── layout.tsx
