기본과제

Phase 1: VNode와 기초 유틸리티

  • core/elements.ts: createElement, normalizeNode, createChildPath
  • utils/validators.ts: isEmptyValue
  • utils/equals.ts: shallowEquals, deepEquals

Phase 2: 컨텍스트와 루트 초기화

  • core/types.ts: VNode/Instance/Context 타입 선언
  • core/context.ts: 루트/훅 컨텍스트와 경로 스택 관리
  • core/setup.ts: 컨테이너 초기화, 컨텍스트 리셋, 루트 렌더 트리거

Phase 3: DOM 인터페이스 구축

  • core/dom.ts: 속성/스타일/이벤트 적용 규칙, DOM 노드 탐색/삽입/제거

Phase 4: 렌더 스케줄링

  • utils/enqueue.ts: enqueue, withEnqueue로 마이크로태스크 큐 구성

  • core/reconciler.ts: 마운트/업데이트/언마운트, 자식 비교, key/anchor 처리
  • core/dom.ts: Reconciliation에서 사용할 DOM 재배치 보조 함수 확인

Phase 6: 기본 Hook 시스템

  • core/hooks.ts: 훅 상태 저장, useState, useEffect, cleanup/queue 관리
  • core/context.ts: 훅 커서 증가, 방문 경로 기록, 미사용 훅 정리

Phase 7: 확장 Hook & HOC

  • hooks/useRef.ts: ref 객체 유지

  • hooks/useDeepMemo.ts, hooks/useAutoCallback.ts: deep 비교/자동 콜백 헬퍼
  • hocs/memo.ts, hocs/deepMemo.ts: props 비교 기반 컴포넌트 메모이제이션

과제 셀프회고

아하! 모먼트 (A-ha! Moment)

  1. Hooks가 조건문 안에서 동작하지 않는 이유를 이해

  • 구현 전에는 훅은 리액트 문서에 나와있는 규칙을 지켜야 한다 정도의 추상적 이해만 있었지만, hooks.state와 hooks.cursor를 직접 만들면서 훅이 순서 기반 상태 조회임을 이해

  • 조건문으로 훅 호출 순서가 달라지면 cursor 기준 상태 매칭이 완전히 무너지고, 같은 컴포넌트라도 전혀 다른 state를 참조하는 심각한 문제가 발생한다는 것을 확인

직접 겪어보니 왜 리액트가 훅의 순서를 강제하는지 자연스럽게 이해


  1. key 기반 매칭 동작 원리

  • reconcile을 구현하는 과정에서 key 기반 매칭이 없으면 불필요한 DOM 조작이 발생하는 걸 알게 됨

  • 단순 index 매칭 같은 경우에는 DOM의 순서가 변경되어도 매번 제거/생성


  1. queueMicroTask로 구현한 batching

  • withEnqueue의 클로저 기반 scheduled 플래그로 여러 번의 render 호출도 한 번만 등록

  • queueMicrotask로 렌더링이 페인트 전에 실행되어 불필요한 중간 렌더링을 모두 줄일 수 있다는 것을 이해


기술적 성장

Virtual DOM과 Reconciliation 이해

  • 프로젝트 전에는 Virtual DOM에 대한 추상적인 이해만 있었지만, 직접 구현해보면서 Reconciliation이 어떻게 동작하는지 명확히 이해.
  • 과제에서는 다루지 않았지만 더블 버퍼링 구조와 렌더/커밋 페이즈 분리를 어떻게 할 수 있을지에 대해 고민

타이밍 기반 버그 추적하는 디버깅 역량 향상

구현 과정에서 다음과 같은 버그들을 겪음

  • 비동기 스케줄링으로 인한 state 업데이트 타이밍 문제
  • cleanup이 실행되지 않는 useEffect
  • deps 비교 누락으로 useMemo/useCallback이 무효화되는 문제
  • 이벤트 중복 등록
  • 여러 setState 호출로 인한 중복 렌더링

해당 문제에 대한 원인을 찾게 되며 동작 방식을 잘 알게됨.


최적화 원리 이해

  • memo, useMemo, useCallback, dom 업데이트 구현 경험을 통해 내부 구조를 이해
  • 불필요한 작업을 안하는 것에 대해 깊게 생각

추가 학습 필요 영역

  • React Fiber의 우선순위 스케줄링
  • 동시성 렌더링 / Suspense 내부 구조
  • React DevTools와 같은 디버깅툴
  • Context API 내부 구조

과제 피드백

  1. 테스트 기반 명세
  • 테스트가 어떤 동작을 구현해야 하는지를 정확히 알려주어 불확실성이 적었다.
  1. 리액트 내부 구조를 이해하게 되는 구조
  • 단순한 이론 이해가 아니라 직접 DOM 조작, diff, 스케줄링을 구현해보면서 내부에서 실제로 어떻게 동작하는지를 이해할 수 있었다.
