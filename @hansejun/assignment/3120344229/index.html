  • 시간표 블록 드래그시 렌더링 최적화

  • 시간표 블록 드롭시 렌더링 최적화

과제 셀프회고

기술적 성장

1. API 호출 최적화

  • Promise.all 내부에서 await를 사용하던 패턴을 제거하여 병렬 호출로 수정
  • 동일 API에 대해 Promise 기반 캐싱 적용
  • SearchDialog 재오픈 시 중복 네트워크 요청 제거
// Before: 순차 실행
await Promise.all([await fetchA(), await fetchB()]);

// After: 병렬 실행
await Promise.all([fetchA(), fetchB()]);

// Promise 캐싱
const createCachedFunction = <T>(fn: () => Promise<T>) => {
  let cache: Promise<T> | null = null;
  return () => cache ?? (cache = fn());
};

효과

  • 초기 로딩 시간 단축
  • 불필요한 API 트래픽 감소

2. Context 구조 개선 (Read / Write 분리)

  • 단일 Context에 혼재되어 있던 state / setter를 분리
  • 읽기 전용 / 쓰기 전용 Context 구성 (CQRS 패턴 적용)
// Read
const SchedulesContext = createContext<SchedulesMap | undefined>(undefined);

// Write
const SetSchedulesContext =
  createContext<React.Dispatch<React.SetStateAction<SchedulesMap>> | undefined>(
    undefined
  );

효과

  • setter만 사용하는 컴포넌트 리렌더링 제거
  • 상태 변경 영향 범위 최소화

3. 콜백 참조 안정화

  • useCallback 의존성 관리 부담을 줄이기 위해 useAutoCallback 도입
  • 콜백 참조는 고정하고, 내부 로직만 최신 상태로 유지
export function useAutoCallback<T extends (...args: any[]) => any>(fn: T): T {
  const ref = useRef(fn);
  ref.current = fn;

  return useCallback(((...args) => ref.current(...args)) as T, []);
}

효과

  • memoized 컴포넌트 불필요한 리렌더링 감소
  • stale closure 문제 방지

4. 연산 및 필터링 메모이제이션

  • SearchDialog 내 필터링 로직을 useMemo로 분리
  • 데이터 변경 시에만 재계산되도록 개선
const filteredLectures = useMemo(
  () => filterLectures(lectures, searchOptions),
  [lectures, searchOptions]
);

효과

  • 검색 / 필터 변경 시 성능 안정화
  • 대량 데이터 기준 연산 비용 감소

5. Drag & Drop 렌더링 최적화

  • DnD 상태를 구독하는 컴포넌트를 Wrapper로 분리
  • Grid, ScheduleBlock을 memo 처리
const ScheduleTableWrapper = ({ active, children }) => (
  <Box outline={active ? '5px dashed' : undefined}>{children}</Box>
);

효과

  • 드래그 중 전체 테이블 리렌더링 제거
  • 다른 테이블에 대한 영향 차단

6. SearchDialog 구조 개선

  • Dialog를 열릴 때만 조건부 렌더링
  • 옵션 / 결과 컴포넌트 분리
  • 결과 Row 단위 memo 적용
{searchInfo && <SearchDialog />}

const LectureRow = memo(({ lecture, onAdd }) => (
  <Tr>
    <Td>{lecture.title}</Td>
    <Td>
      <Button onClick={() => onAdd(lecture)}>추가</Button>
    </Td>
  </Tr>
));

효과

  • Dialog 비활성 상태에서의 불필요한 연산 제거
  • 무한 스크롤 시 기존 Row 리렌더링 방지

7. 모달 Lazy 렌더링

  • 삭제 확인 / 검색 모달을 Lazy 렌더링
  • 실제 상호작용 시점에만 콘텐츠 생성
<Popover isLazy>
  <PopoverTrigger>{/* trigger */}</PopoverTrigger>
  <PopoverContent>{/* modal content */}</PopoverContent>
</Popover>

효과

  • 초기 렌더링 비용 감소
  • 메모리 사용량 절감

코드 품질

  • useAutoCallback 훅: 재사용 가능한 유틸리티로 프로젝트 전반에 적용 가능
  • Context Read/Write 분리: 일관된 패턴으로 ScheduleContext, SearchInfoContext에 동일하게 적용
  • DnD Wrapper 분리: children 패턴을 활용해 렌더링 범위를 명확히 제한

학습 효과 분석

1. 렌더링 최적화 이해 단순히 memo, useMemo, useCallback을 사용하는 것이 아니라, 왜 리렌더링이 발생하는지 원인을 파악하는 것이 중요하다는 것을 배웠습니다.

2. Promise.all의 올바른 사용법

Promise.all 내부에서 await를 사용하면 배열 생성 시점에 이미 순차적으로 실행된다는 것을 명확히 이해하게 되었습니다.

3. CQRS 패턴의 프론트엔드 적용

CQRS 패턴을 React Context에 적용하여 읽기/쓰기를 분리함으로써 구독 최적화가 가능하다는 것을 배웠습니다.

4. 컴포넌트 분할 전략

성능 최적화 관점에서 컴포넌트를 분할하는 기준을 배웠습니다.

  • 상태 구독 범위가 다른 요소는 분리
  • 자주 변경되는 상태를 구독하는 부분을 격리
  • children 패턴을 활용한 리렌더링 차단
