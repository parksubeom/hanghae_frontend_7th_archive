8.hook이란?

  • 함수 컴포넌트에서 상태 관리 및 생명주기 기능(lifecycle features)을 사용할 수 있게 해주는 도구
  1. 렌더 스케줄링이란?
  • 상태가 바뀔 때마다 곧바로 렌더링하지 않고, 여러 업데이트를 모아서 가장 효율적인 타이밍에 렌더링하도록 조절하는 기술
  • 왜 필요해? >>상태가 연속으로 변화 = 리렌더링 연속발생, 리렌더링은 비용이 크다 >>>최적의 시점에 묶어서 처리(렌더 스케줄링)
- 기본규칙
1. 리액트의 기본 기능
상태 관리 (useState)
생명주기 처리 (useEffect)
리렌더링 제어 (useMemo, useCallback)
DOM 접근 (useRef)
2. 반드시 ‘use’로 시작하는 함수 이름이어야 함
→ 이 규칙을 지켜야 리액트가 내부적으로 Hook 호출을 추적할 수 있어.
4. 컴포넌트 렌더링 과정에서 호출되어야 함
→ 조건문, 반복문 안에서 호출하면 안 되는 이유도 이것 때

한줄요약 : 로직을 만드는데 리액트의 기본기능이 들어가 있는 재사용가능한 함수, 
+ Hook은 “기능 자체”가 아니라 “기능을 호출하는 API”

나를 위한 예시
“가스레인지에 불이 나온다” = 기능
“점화 버튼” = 그 기능을 사용하기 위한 방법
Hook은 점화 버튼 같은 존재야.

학습 효과 분석

[과제 이해] 혼란: react를 만들어보는데 virtual Dom이랑 VNode를 왜 사용하고 왜 제작하는거지? 해결: React는 화면을 그릴 때 JSX → VNode → 실제 DOM 순서로 처리 => 그래서 이번 과제의 목표는 JSX를 VNode로 변환해보는 거라고 이해.

### 리액트의 깊은 이해
리액트의 정의: “리액트는 복잡한 UI를 상태 변화에 따라 자동으로 효율적으로 다시 그리기 위해 Virtual DOM 기반으로 컴포넌트를 렌더링하고 필요한 부분만 업데이트하는 라이브러리다.”

 1. React 내부
        - JSX를 React.createElement() → VNode 객체로 변환
        - VNode 트리를 Virtual DOM으로 관리

 2. 렌더링/상태 변화 시
        - 새로운 VNode 트리 생성
        - 이전 VNode 트리와 비교(diff)
        - 실제 DOM에는 최소한만 반영

이러한 일을 해주는게 리액트이다.

0단계 기초

VNode란?

  • DOM 요소를 프레임워크가 JS 객체로 추상화해둔 것

기본 틀

type VNode = {
  type: string | symbol | ComponentType; // 요소 타입
  props: {
    children?: VNode[]; // 자식 노드들
    [key: string]: any; // 기타 속성
  };
  key?: string | number; // 리스트 렌더링용 키
};

예시:

// JSX
<div className="container">
  <h1>Hello</h1>
  <p>World</p>
</div>

// VNode로 변환
{
  type: "div",
  props: {
    className: "container",
    children: [
      { type: "h1", props: { children: ["Hello"] } },
      { type: "p", props: { children: ["World"] } }
    ]
  }
}

1단계

VNode와 기초 유틸리티 >>> React의 Virtual DOM과 JSX 변환 메커니즘

질문? DOM을 어떻게 비교해서 바꿔치기는하는가? -> 얇은 비교, 깊은 비교 함수를 제작 === shallowEquals, , deepEquals -> 왜 나눠서 함수를 제작하는가? === useEffect는 얇은 비교 / useDeepMemo,deepMemo 일때 깊은 비교

돔을 교체할 때 얕은 비교를 해서 감지후 변경하는데 특별한 경우에는 깊은 비교를 할때도 있다.

질문? DOM에는 여러 컴포넌트가 있는데 어떻게 해당 컴포넌트를 찾아서 교체함? -> 컴포넌트 고유 경로를 생성 함수 제작=== createChildPath(parentPath, index, key) -> 왜 === 같은 컴포넌트가 같은 페이지에 있을 때 각각 작동하게하기 위해서.

컴포넌트마다 고유한 경로를 가짐, 경로를 통해 훅 상태 격리, 같은 컴포넌트라도 위치가 다른 상태를 가짐
