정리

  1. 멘탈 모델 : 시스템이나 개념을 이해하고 예측하기 위해 머릿속에 그리는 구조적 이미지/틀
  2. FSD 기능중심 아키텍처: 기능을 중심으로 응집도를 높인다. => 규모가 클 때 빛을 발휘한다.
  3. 관심사를 분리하는 이유 : 작은 프로젝트의 느낌으로 개발하기 위해.
  • 관심사를 어떤 기준으로 나눌 수 있을까? -> 역할중심(화면디자인, 인터렉션 서버와 데이터 통신, 데이터 변화 등등)
  • app : 애플리케이션의 전체적이 설정과 초기화 담담
  • pages: 실제 페이지를 구성하는 컴포넌트
  • widgets: 재사용 가능한 복잡한 UI 블럭(features + entity를 사용해서 제작한 것들)
  • features: 특정 비즈니스 기능을 담당(변경 o)
  • shared: 공통으로 사용되는 유틸리티와 UI 블럭
  • entity: 순수 함수 로직 (변경 x)

FSD 한줄 정리 :

단품 (shared) + 세트 구성품 (entity) + 세트(features) + 패키지(wegets) + 주문서(단품부터 패키지까지 내가 원하는데로 구매가능)(page)

  1. shared (데이터 없는 UI + 공통 로직)
  • button
  • input
  • textarea
  • select
  • modal : radix-ui/react-dialog를 이용해서 shared에 넣었다.
  • table
  1. entity (데이터 받을 수 있는 UI + 도메인 데이터)
  • comment
  • post: 게시글/ 댓글리스트에서 사용되는 버튼들
  • tage
  • user
  1. features
  • comment (비즈니스 로직 + UI)
  • post: 리스트에서 사용되는 edit 버튼의 기능들
  • tag: 리스트 안에서 작동하니 posttablerowwithclick과 함께 넣어줬다.
  • user: 사용자는 데이터들을 갖고 있다
  1. widgets(위의 세가지를 조합해서 조합)

의미: Widget은 Features, Entity, Shared 모두 사용 가능 Features는 Entity, Shared 사용 가능 Entity는 Shared만 사용 가능 Shared는 아무것도 사용하지 않음

나만의 룰

  1. shared = 쌩 퍼블 걍 그데로 박아도 되는 것들 텍스트의 변화
  2. entity = 데이터가 들어갈 수도 있는 퍼블 즉 텍스트의 값들이 변화가 될 수도 있는 ui
  3. features = 함수(비즈니스 로직)로 인해서 데이터가 변화되는 퍼블 반문
  • button은 button마다 텍스트가 변하잖아요 => 하지만 처음 제작할 때 버튼 텍스트를 집어 넣잖아. 그건 데이터가 아니잖아.
  1. widgets = 완성된 ui

프론트엔드의 두 종류의 상태

  1. 클라이언 상태 =>jotai
  • 모달 열림?
  • 검색어 입력함?
  • 선택한 탭

  1. 서버 상태 => TanStack Query
  • 게시물 목록
  • 댓글 리슽
  • 사용자 정보
  • 좋아요 수 => 서버에서 가져오는 데이터, 서버가 변경되면 새로 가져와야함.

한줄 요약:

  • 둘의 상태는 연결된 것이 아닌 '역할 분리' 이다.
  • 클라이언 상태는 내가 컨트롤 가능
  • 서버 상태는 내가 컨트롤 불가능

상태는 왜 필요해?

  1. jotai
  • 리액트는 단방향성이기 때문에 컴포넌트끼리의 상태공유가 번거러움
  1. TanStack Query
  • 서버의 데이터를 받아서 ui로 보여줘야 한다. (서버에서 무슨 일이 일어나는지 나는 모름 -> 나의 영역이 아님)
  • 하지만 서버 데이터에 오류가 났다고 내 브라우저 ui에 빈화면을 보여줄 수는 없다.
  • 서버 데이터의 불확실성에 대비해야하는 ui가 필요하다. 이것을 하기위해서 TanStack Query가 필요한 것이다. ( 예를들어
    1. 서버 데이터를 가져올때 얼마나 걸릴지 모름 >> 이때 ui에서는 로딩을 보여줘야함,
    2. 네트워크가 끊겨 서버데이터가 날라 갔을 경우 >> ui에서는 error를 보여줘야함. )

import { useAtom } from "jotai";
import { searchAtom } from "./state";
import { usePosts } from "./usePosts";

function PostList() {
  const [search] = useAtom(searchAtom);
  const { data, isLoading } = usePosts(search);

  if (isLoading) return <p>로딩 중...</p>;

  return (
    <ul>
      {data.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
}



usePosts.jsx
import { useQuery } from '@tanstack/react-query';

export function usePosts(search) {
  return useQuery({
    queryKey: ['posts', search],
    queryFn: () => fetch(`/api/posts?search=${search}`).then(res => res.json())
  });
}
