과제 체크포인트

배포 링크

과제 요구사항

  • 배포 후 url 제출

  • API 호출 최적화(Promise.all 이해)

  • SearchDialog 불필요한 연산 최적화

  • SearchDialog 불필요한 리렌더링 최적화

  • 시간표 블록 드래그시 렌더링 최적화

  • 시간표 블록 드롭시 렌더링 최적화

과제 셀프회고

기술적 성장

회사에서는 거의 새로운 기능 구현 업무만 하다보니 성능 최적화를 해본 경험이 많이 없었다. 이번 과제를 하면서, 메모이제이션 또는 적절한 상태 관리를 통해 얻을 수 있는 성능상의 이점 등을 명확하게 확인할 수 있었다. 또한, 자주 사용하지 않던 React DevTools을 가지고 놀면서 실무에서 잘 활용해 봐야겠다는 생각이 들었다.

학습 효과 분석

  • React DevTools을 활용한 디버깅 능력
  • 메모이제이션 기법으로 얻을 수 있는 성능 향상 정도 인지
  • 최악의 리렌더링 상황 등

과제 피드백

  • 실무에서 잘 다루지 않는 성능 최적화를 해볼 수 있어서 좋았습니다.
  • 에타와 비슷한 UI가 익숙해서 재밌었습니다.

리뷰 받고 싶은 내용

  • React DevTools의 Profiler나 lighthouse 외에 실무에서 성능 측정에 추가로 사용하시는 도구가 있으신가요?
  • 성능 최적화와 유사한 작업들을 할 때, 문서화하는게 늘 고민인거 같아요. 코치님께서는 성능 최적화 등의 작업을 진행하시기 전, 후에 문서화를 하여 팀에 공유하시나요?
  • 언제 & 어떤 상황에서 문서화를 하시는지 궁금합니다!

과제 피드백

안녕하세요 준태님! 마지막 과제 잘 진행해주셨네요. 역시 믿고 보는 준태님의 과제입니다 ㅎㅎ

React DevTools의 Profiler나 lighthouse 외에 실무에서 성능 측정에 추가로 사용하시는 도구가 있으신가요?

흠.. 저는 크롬 개발자 도구에서 제공하는 성능탭, 네트워크탭 말고는 별도의 도구를 사용하진 않는 것 같아요. 아마 센트리 정도..? 센트리가 자연스럽게 Web Vitals에 대한 추이/추세를 볼 수 있도록 로그를 쌓아줬어요 ㅋㅋ

성능 최적화와 유사한 작업들을 할 때, 문서화하는게 늘 고민인거 같아요. 코치님께서는 성능 최적화 등의 작업을 진행하시기 전, 후에 문서화를 하여 팀에 공유하시나요?

팀원들이 함께 보면 좋을 내용의 경우 대체로 문서화를 했었어요. 특히 요즘에는 AI가 어느정도의 내용은 커버해주다보니 큰 맥락 정도만 만들어서 공유해주면 AI가 꽤 잘 만들어준답니다 ㅎㅎ 가령, git diff 같은걸 토대로 만들어본다거나!?

언제 & 어떤 상황에서 문서화를 하시는지 궁금합니다!

단순히 git log나 pr 등으로는 커버하기 어려운 내용에 대해서 문서화를 하기도 하고 일을 진행하게된 "원인"에 대한 이해가 필요한 경우에 문서화를 하기도 했어요.

그런데 이건 내가 하는 일의 맥락을 얼마나 디테일하게 공유하고 싶은지에 따라 달라지는 것 같네요 ㅎㅎ 위에서도 언급했지만, AI가 있다보니 이제는 문서화에 대한 부담이 거의 없어진 상태여서, 단순 이슈 해결이 아니라면 대체로 문서화를 하려고 하는 편입니다.

토스 같은 회사의 경우 "빠르게 실패하고 나아가기"가 목표라서, 이 경우에는 오히려 일부로 문서화를 하지 않는다고 해요. 어차피 결과가 확실하지 않으면 빠르게 폐기될 예정이니까요 ㅎㅎ

회사마다 일하는 스타일이 다르기 때문에 이에 대한 고려도 필요하답니다~!
