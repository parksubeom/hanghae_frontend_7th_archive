과제 피드백

고생하셨습니다 재윤님! 회고도 아주 구체적으로 잘 작성해주셨었는데요. 학습하셔야 되는 부분부분들을 구체적으로 잘 짚어 주신것같아요 :+1

static-site-generate.js에서 SSG 빌드 시 모든 상품 페이지를 순차적으로 생성하고 있습니다. 현재는 약 20개 상품으로 테스트 중인데, 상품이 1000개 이상으로 늘어났을 때 빌드 시간과 메모리 사용량이 걱정됩니다.

정확합니다! 이게 SSG의 단점과도 이어지는데요. Promise all을 쓰든 결국 절대적인 페이지가 늘어나게 되면 쉽게 해결이 되는 문제가 아닐거에요. 컴퓨팅 파워도 제한적일거구요. 목적에 맞춰 각각 다른 페이지들을 어떤건 SSG, 어떤건 다른 패턴을 적용하는것도 방법일 수 있지만, 선택적 빌드(변경만 있는 것들만 한다라거나..)등등.. 방법을 찾아서 해결하는 편이 좋을거에요. 이 고민이 꽤나 오래된걸로 알아서 SSG를 제공하는 여러 프레임워크들이나 글들을 보면 관련 고민을 어떻게 풀었는지 알 수 있지 않을까..! 싶습니다.

ssrContext.ts에서 전역 변수 let ssrContext를 사용하여 요청별 상태를 관리하고 있습니다.

음.. 정확한 답변일지는 모르겠지만, 아마 말씀해주신거처럼 해당 위험이 있을 것 같아요. AsyncLocalStorage같은 걸 활용해서 요청별 컨텍스트를 분리하거나 express 미들웨어서 별도 처리를 해주는 형태로 분리할 수 있을 것 같은데.. 해당 부분에 대한 고민은 필요할 것 같습니다!

고생하셨고 마지막 주차도 화이팅입니다!
