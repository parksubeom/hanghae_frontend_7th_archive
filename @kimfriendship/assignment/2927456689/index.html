[2팀 김우정] Chapter 1-1. 프런트엔드 테스트 코드 익숙해지기

by kimfriendship2025.10.19

HARD

과제 체크포인트

기본과제

  • 총 11개의 파일, 115개의 단위 테스트를 무사히 작성하고 통과시킨다.

질문

Q. handlersUtils에 남긴 질문에 답변해주세요.

테스트가 병렬로 실행되더라도 문제가 발생하지 않으려면, 각 테스트 환경이 서로 영향을 주지 않도록 보장해야 합니다. 각 테스트 케이스에서 동일한 handlers를 사용할 경우, 모두 같은 데이터를 참조하게 되어 충돌이 생길 수 있습니다. 그래서 handlerUtils를 활용해 각 테스트가 독립적인 환경 안에서만 event 데이터를 조작하도록 구현했습니다.

Q. 테스트를 독립적으로 구동시키기 위해 작성했던 설정들을 소개해주세요.

  • handlersUtils 작성 각 테스트마다 독립적인 handlers와 event 데이터를 생성하도록 구성했습니다.
  • beforeEach / afterEach 활용 각 테스트 실행 전후로 공용 상태를 초기화했습니다.
  • 타이머 제어 (vi.useFakeTimers) 타이머 기반 로직(setTimeout, setInterval 등)이 있는 테스트는 가상 타이머를 사용했습니다.
  • 테스트 데이터 분리 각 테스트 케이스에서 사용하는 이벤트 데이터(event)를 별도로 정의하여, 데이터 간 간섭을 방지했습니다.

심화 과제

  • App 컴포넌트 적절한 단위의 컴포넌트, 훅, 유틸 함수로 분리했는가?
  • 해당 모듈들에 대한 적절한 테스트를 2개 이상 작성했는가?

과제 셀프회고

기술적 성장

  • 테스트 코드를 직접 작성해보고 싶다는 생각은 늘 있었지만, 실제로 도입하기 어려워 미뤄왔습니다. 이론적인 공부만으로는 한계가 있다고 느꼈는데, 이번 과제를 통해 실제로 작성해보면서 많은 것을 체득할 수 있었습니다.

  • 테스트 코드를 잘 짠다는 것은 단순히 “코드를 잘 짠다”와는 다른 영역이라는 걸 느꼈습니다. 기본적인 코드 완성도뿐 아니라, 의미 있는 테스트 케이스를 정의하고, 각 테스트를 순수하게 검증할 수 있도록 구성하는 것이 얼마나 중요한지 깨달았습니다.

코드 품질

코드 품질은 솔직히 말씀드리면 부끄러울 정도로 엉망이라고 생각합니다.. 😭 각 테스트에서 사용하는 event 객체들도 정리가 부족했고, handlersUtils 역시 더 간결하게 만들 수 있었을 것 같습니다. 문서를 충분히 참고하지 않고 작성한 부분이 많아, 올바르지 않게 사용한 코드도 꽤 있을 것 같습니다. TODO로 남겨둔 고민거리나 개선 포인트들도 아직 다 정리하지 못했지만, 기록해둔 부분들은 다시 돌아보면서 꼭 보완해보고 싶습니다.

학습 효과 분석

과제 피드백

리뷰 받고 싶은 내용

  • 이벤트 객체들을 일일이 써주는 게 가장 귀찮기도 하고, 테스트 케이스마다 조금씩 달라지는 프로퍼티 값들을 어떻게 가독성있게 보여줄까 고민이 되었습니다. 리팩토링을 결국 못했지만 추천하는 방법이 있으시면 알려주시면 감사하겠습니다. (중간에 getDummyEvent 유틸함수를 만들어서 필요한 값만 override되게 할까도 생각만 해봤습니다.)

  • 통합테스트 중에 notification querying을 끝까지 못했었는데요, 다른 분들은 어떻게 해결하셨는지 궁금합니다!

  • 저는 parseDateTime 테스트에 대해서 필요한가 의문이 들었는데 다른 분들은 어떻게 생각하셨는지 궁금합니다. parseDateTime은 Date 함수를 사용하는 것밖에 없어서, 검증하려는 값을 만들 때 똑같이 Date를 쓰는 거밖엔 없는 것 같아서 굳이 필요한 테스트일까 생각했습니다.

궁금한 점

회사 프로젝트에도 테스트를 도입해보고 싶은데, 다국어 서비스(i18n) 환경에서는 어떤 점을 주의해야 할까요? 텍스트 검증이 필요한 경우, setup에서 언어를 하나로 고정한 뒤 해당 언어 기준으로만 검증하는 방식이 적절할지 궁금합니다.

과제를 수행하면서 느낀점

과제 시작 전 생각

진짜 너무너무 막막했고, 한 파일 한 파일 열어가면서 진짜 새로운 벽이 나타나서 좌절의 연속이었습니다.. 과제 다 끝낼 수 있을까? 이렇게 10주를 어떻게 버티지? 이런 생각 수도 없이 한 것 같습니다. 그리고 과제를 진행하면서도, 로직이 그렇게 어려운 코드를 짜는 것도 아닌데 나는 왜 이 테스트 하나 작성하는데 몇 시간을 쓰고 있을까 하는 자괴감이 많이 들었습니다.

과제 제출 후 생각

그래도 어찌저찌 과제를 끝내서 다행이라는 생각입니다. 테스트 케이스 전체를 통과했을 때의 그 짜릿함이란...🤩 시간이 더 있었다면 리팩토링도 조금 더 하고, 고민되던 부분들에 대해 리서치도 해보고 싶었는데 그러지 못해 아쉽습니다. 또 시간에 쫓기다 보니 학습 문서나 자료를 충분히 참고하지 못한 채 과제를 진행하게 되었는데, 개념을 좀 더 명확히 잡고 시작했다면 시행착오를 줄일 수 있었을 것 같아요. 다음 주에도 비슷한 어려움이 반복될 것 같아 살짝 불안하긴 하지만 허허, 그래도 꾸준히 복습하면서 감을 잡아보려고 합니다!
