Props 분리 기준 세우기

어떤 props를 남기고 어떤 props를 제거할지 기준을 세우는게 고민이었습니다.

남긴 props (도메인 데이터)

  • ProductCard의 product - 어떤 상품인지 명시
  • CartItem의 item - 어떤 장바구니 아이템인지 명시
  • ProductList의 searchTerm - UI 상태

제거한 props (콜백 핸들러, 파생 데이터)

  • onAddToCart, onRemoveFromCart - hooks에서 직접 호출
  • remainingStock - hooks에서 계산
  • selectedCoupon - 전역 상태에서 직접 참조

결론적으로 "이 컴포넌트가 뭘 렌더링하는지 명시하는 데이터"는 props로 남기고, "어떻게 동작하는지"는 전역 상태와 hooks로 처리하는 방향으로 정리했습니다.

폴더 구조

src/advanced/
├── atoms/          # Jotai 전역 상태 (도메인별 분리)
├── hooks/          # 커스텀 훅 (Adapter 패턴)
├── models/         # 순수 계산 함수 (비즈니스 로직)
├── features/       # 도메인별 컴포넌트
│   ├── cart/
│   ├── product/
│   ├── coupon/
│   ├── order/
│   └── admin/
├── pages/          # 페이지 컴포넌트
├── components/     # 공통 UI 컴포넌트
├── constants/      # 상수, 초기값
└── utils/          # 유틸리티 함수

atoms → hooks → features 순서로 의존성이 흐르도록 구조를 잡았습니다.
