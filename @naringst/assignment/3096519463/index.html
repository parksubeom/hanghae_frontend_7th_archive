처음에 발제를 들었을 때, 준일 코치님께서 서버에서 js 파일을 가져와서 ~~를 처리한다~ 이렇게 말씀해주셨습니다. 그런데 서버에서 클라이언트 데이터를 어디서 어떻게 가져오는거지?? 라는 의문이 들었습니다. 따라서

클라이언트 코드 작성 -> 빌드 -> 서버에서 클라이언트 빌드된 코드 가져오기 -> 서버 처리 -> 다시 브라우저 렌더링 과정

에 대해 정리해보았습니다.

질문 주신 것들 보면 이미 방향을 알고 계신 것 같아요. 렌더링 직전에 setServerUrl 다시 호출하는 건 임시방편이고, 말씀하신 대로 요청별 컨텍스트 분리가 근본적인 해결책이에요. router를 재사용하기 보다는 요청마다 새 인스턴스를 만들어 활용해보세요.

productStore도 마찬가지예요. 지금은 페칭 시간이 짧아서 문제가 안 드러나는 거지 요청 많아지면 섞일 수 있어요. useSyncExternalStore 활용은 적절하게 잘 하셨습니다
