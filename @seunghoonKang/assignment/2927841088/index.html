과제 체크포인트

심화 과제

HARD

과제 체크포인트

기본과제

  • 총 11개의 파일, 115개의 단위 테스트를 무사히 작성하고 통과시킨다.

질문

Q. handlersUtils에 남긴 질문에 답변해주세요. Q. 테스트를 독립적으로 구동시키기 위해 작성했던 설정들을 소개해주세요.

심화 과제

  • App 컴포넌트 적절한 단위의 컴포넌트, 훅, 유틸 함수로 분리했는가?
  • 해당 모듈들에 대한 적절한 테스트를 2개 이상 작성했는가?

과제 셀프회고

궁금한 점

  • 뭘 알고 뭘 모르는지와 같은 상태 같습니다. 리뷰를 받고 싶은데 어떤걸 어떻게 받고 물어야할 지 사실 감이 안와서, 스스로 더 작성해보고 익숙해져봐야하지 않나 싶습니다. 더 친해지는 좋은 방법이 있다면 알려주시면 감사하겠습니다.

과제를 수행하면서 느낀점

테스트 코드가 처음이라 여러 부분에서 많이 애를 먹었던 한 주였습니다.. 꽤나 시간을 투자했다 생각함에도 여전히 작성에 막히는 부분이 많습니다. 테스트 코드 자체가 새로 학습한 개념으로 볼 수 있다 보니, 짜면서 차츰 감을 잡아가려 하였습니다. 아쉽게도 기간 내에 통합테스트까지 하지 못했습니다.

성장했다 느끼는 부분은, 아직은 느껴지지 않습니다. 테스트 코드란 게 무엇인지는 알겠는데, 여전히 왜 써야하는가? 에 대한 고민이 됩니다. 이는 제가 여전히 활용하는 데 애먹고 있다는 것 같은데, 조금 더 테스트 코드와 친해져야 할 것 같습니다.

과제 시작 전 생각

Hard로 시작해도 열심히 하면 완료할 수 있지 않을까? (못했습니다)

과제 중 생각

이래저래 고민을 많이 했던 건 처음 시작했던 easy.useCalendarView 파일이었습니다. currentDate는 오늘 날짜인 "2025-10-01"이어야 한다 라는 테스트에, 오늘 날짜는 10월 00일인데, 이 날짜가 바뀔때마다 어떻게 검증하지? 생각하며 작업했던 것 같습니다. 작성하다보니 추후에는 mockData를 미리 만들어놓고 사용한다는 걸 알게 되었는데, 이러한 부분도 mockData 처럼 처리했으면 됐을까? 생각했습니다.

기타

기술적으로 작성한 내용이 부족하네요. 고민이 많은 첫 주인 것 같습니다.
