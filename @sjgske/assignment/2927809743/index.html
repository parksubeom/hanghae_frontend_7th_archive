질문

Q. handlersUtils에 남긴 질문에 답변해주세요.

Q. 테스트를 독립적으로 구동시키기 위해 작성했던 설정들을 소개해주세요.

handlers.ts 에서 핸들러를 생성하는 팩토리 함수 createHandlers 를 만들었습니다. 해당 함수를 호출할 때마다 새로운 독립적인 handler이 생성되도록 하기 위함입니다. 각 handler는 자체 Map을 클로저로 가지고 있어서 상태가 격리됩니다. seupTests.ts의 afterEach에서 resetHandlers로 초기화시킨 후, createHandlers() 호출하여 핸들러를 교체하여 각 테스트가 동일하게 초기 데이터 Map()를 참조할 수 있도록 하였습니다.

afterEach(() => {
  server.resetHandlers();
  server.use(...createHandlers());
  vi.clearAllMocks();
});

리뷰 받고 싶은 내용

createHandlers를 공통으로 관리하기 편하도록 setupTests.ts 파일에 집어넣고 보니 확장성이 떨어지는 것 같다고 느껴졌습니다. 코치님께서 handlerUtils.ts에 템플릿으로 남겨주신 방식대로 메소드별로 작성하여 테스트 시마다 넣어주는 것과 공통으로 넣어주는 것 어떤 방식이 더 적절할지 리뷰 요청드립니다.

통합테스트 시 요소를 찾는 부분(getBy, findBy, queryBy..)을 시간 이슈로 제대로 학습하지 못하고 그냥 넣어보고 테스트 통과하면 이건가보다..하는 식으로.. 했어서 테스트코드가 통일성이 없는데(ex. 어떤 것은 testId로 찾고, 어떤 것은 role, labelText 등 중구난방), 어떤 기준으로 요소 찾는 메서드를 선택하고 통일성 있게 해야 할지 리뷰 부탁드립니다.

통합테스트 시 일정 등록, 수정, 삭제와 같은 api 호출이 있는 기능은 userEvent로 테스트 해야 할까요(ex. 버튼 클릭)? 아니면 server.use() 처럼 api 호출을 하는 것이 좋을까요? 리뷰 부탁드립니다.
