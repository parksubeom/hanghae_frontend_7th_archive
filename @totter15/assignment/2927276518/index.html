과제 체크포인트

기본과제

  • 총 11개의 파일, 115개의 단위 테스트를 무사히 작성하고 통과시킨다.

질문

Q. handlersUtils에 남긴 질문에 답변해주세요.

이벤트에 생성, 수정이후 조회를 하게되면 msw로 할당한 mock배열이 생성이든, 수정이든 동일하게 호출이 됩니다. 이를 해결하기 위해서 handlerUtils함수 내부에서 사용되어질 events 배열을 만들고, post나 put요청시에 해당 배열을 변경하여 변경된 배열을 get호출시에 return 시켜줄 수 있도록 만들었습니다.


export const setupMockHandlerCreation = (initEvents = [] as Event[]) => {
  const events = [...initEvents];

  server.resetHandlers();
  server.use(
    //post에 넣은 event 가져오기
    http.post('/api/events', async ({ request }) => {
      const newEvent = await request.clone().json();
      events.push(newEvent);
    }),

    //get에 요청시 가져온 event넣어서 반환
    http.get('/api/events', async () => {
      return HttpResponse.json({ events });
    })
  );
};

Q. 테스트를 독립적으로 구동시키기 위해 작성했던 설정들을 소개해주세요.

test를 독립적으로 구동시키기위헤 각 handlerUtils에 server.resetHandlers를 통해서 항상 reset된 상태에서 핸들러가 동작할 수 있게 만들어 주었습니다.

export const createEvent = ({

과제 피드백

단기간에 많은 코드를 작성하면서 좀더 빠르게 지식을 습득할 수 있었습니다. 학습하기에 일단 많이 해보기는 좋은 방식중 하나인것 같습니다. 모호한 테스트들에 대한 예시를 직접경험할 수 있어서 좋았습니다. 단위, 통합, api 모킹같이 실제로 바로 실무에 적용할 수 있는 예제들을 직접 구현해 볼수 있어서 실제 실무에서도 적용해볼 수 있을 것 같습니다.

리뷰 받고 싶은 내용

1.

테스트코드는 명세의 역할을 할수 있다고 알고있습니다. 그렇다면 테스트 작성시에 함수의 반환값을 그대로 비교하는 것이 좋은 방식인지, 아니면 올바른 조건을 찾아내서 그 조건을 검증하는것이 좋은 방식인지가 궁금합니다.

// 함수의 반환값을 그대로 비교하는 방식
it('월의 마지막 날짜를 포함한 주를 올바르게 처리한다', () => {
const date = new Date('2025-10-31');
	expect(getWeekDates(date)).toEqual([
		new Date('2025-10-26'),
		new Date('2025-10-27'),
		new Date('2025-10-28'),
		new Date('2025-10-29'),
		new Date('2025-10-30'),
		new Date('2025-10-31'),
		new Date('2025-11-01'),
	]);
});

// 조건을 찾아내서 그 조건을 검증하는 방식
it('getWeekDates는 입력된 날짜의 요일의 index에 해당 날짜가 있고, 총 7개의 날짜를 반환한다', () => {
	const date = new Date('2025-10-31');
	const dateIndex = date.getDay()

	assertDate(weekDates[dateIndex], date);
	expect(weekDates).toHaveLength(7);
});

예를들어 월의 마지막 날짜를 포함한 주를 올바르게 처리한다라고했을때, 함수의 return 값을 그대로 내려주는게 처음보는 사람이 함수를 이해하려고 할때 이해하기가 쉬울것 같습니다. 작성된 조건을 좀더 특정할 경우 qna시간에 말씀을 해주신것처럼 문제 발생시 어디 부분이 문제가 되는건지 바로알기가 쉬울것 같습니다.

1-1. 이번에 제출한 코드에서는 검증값을 단순하게 보여주는게 더 좋을 것 같다는 생각에 첫번째 방식을 사용했는데, 아래의 방식도 괜찮은 방법인지가 궁금합니다.

1-2. 그리고 두번째 방법을 사용했을경우 getWeekDates함수는 입력된 날짜가 제대로된 index에 들어가고, 7개의 date만 제대로 반환되는지만 확인하게 되기에, 여기서 월의 마지막 날짜 혹은 주의 마지막 날짜등의 검증은 불필요하게 느껴집니다. 두번째 방법을 사용시 월의 마지막 날짜 혹은 주의 마지막 날짜등의 검증은 생락이 가능한건지도 궁금합니다.

2.

hadlersUtils에서 제가 사용한 방식이 맞는 방식인지가 궁금합니다. http.post핸들러에서 요청한걸 가로채는(?) 방식을 사용했는데, 현재 코드상에서는 post요청의 응답에 영향이 가지는 않을 것 같긴합니다. 하지만 요청 중간에 어떤 동작을 넣은지라 해당 코드가 이후에 다른 누군가에 의해 수정되거나 하면 기존에 보내는 post 요청의 응답에 문제가 생길 수도 있지 않을까 하는 찝찝함이 있는 것 같습니다.

 const setupMockHandlerCreation = (initEvents = [] as Event[]) => {
  const events = [...initEvents];

  server.resetHandlers();
  server.use(
    //post에 넣은 event 가져오기
    http.post('/api/events', async ({ request }) => {
      const newEvent = await request.clone().json();
      events.push(newEvent);
    }),

    //get에 요청시 가져온 event넣어서 반환
    http.get('/api/events', async () => {
      return HttpResponse.json({ events });
    })
  );
};
